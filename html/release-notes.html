Search service 12.0.0
8.03.2022

 What's Changed

 Breaking Changes ðŸ”¥

  • Add equalize configuration to MultiMatchSearch feature by @robertoaz in https://github.com/empathyco/search-service/pull/471 This outside object of equalize will not longer be used by feature:
{
 "endPoints": [],
 "equalize": [] // <- NOT LONGER USED
}

Also deprecate the "fields" object inside the "MultiMatchSearch" feature:

{
 "name": "multiMatchSearch",
 "config": {
 "fields": [] // <- DEPRECATED
 }
}

Now the weights of the searchable fields will be added in a new object "searchableFields" inside "MultiMatchSearch" feature. Example:

{
 "name": "multiMatchSearch",
 "config": {
 "searchableFields": [
 {
 "name": "description",
 "boost": 3
 },
 {
 "name": "brand"
 }
 ]
 }
}

Check documentation updated on here: https://github.com/empathyco/search-service/blob/main/docs/features/multi-match-search/multi-match-search-feature.md

Apart from that, new "playEqualizeProvider" has been added, that is injected to the "MultiMatchSearch" feature, instead of the use of the "equalizeProvider", that has been deprecated too.
