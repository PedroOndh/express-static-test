NextQueries

 NextQueries

Simple next-queries component that renders a list of suggestions, allowing the user to select one of them, and emitting the needed events. A next query is a suggestion for a new search, related to your previous query. I.e. If people normally search for shirts, and then trousers, trousers would be a next query of shirts.

 Props

Name Description Type Default
animation Animation component that will be used to animate the suggestions. Vue
maxItemsToRender Number of next queries to be rendered. number
highlightCurated Flag to indicate if the curated next queries should be displayed different. boolean false

 Slots

Name Description Bindings
(name - type - description)
suggestion Next Query item suggestion Suggestion - Next Query suggestion data
highlightCurated boolean - True if the curated NQs should be highlighted
index number - Next Query suggestion index
suggestion-content Next Query content suggestion Suggestion - Next Query suggestion data
shouldHighlightCurated boolean - True if the curated NQ should
be highlighted
index number - Next Query suggestion index

 Examples

 Basic example

You don't need to pass any props, or slots. Simply add the component, and when it has any next queries it will show them

<NextQueries />

The component has three optional props. animation to render the component with an animation, maxItemToRender to limit the number of next queries will be rendered (by default it is 5) and highlightCurated to indicate if the curated Next Queries inside the list should be highlighted.

<NextQueries
  :animation="FadeAndSlide"
  :maxItemsToRender="10"
  :highlightCurated="true"
/>

Overriding Next Queries' Content

You can use your custom implementation of the Next Query's content. In the example below, instead of using the default Next Query's content, an icon is added, as well as a span with the query of the Next Query suggestion.

<NextQueries>
  <template #suggestion-content="{suggestion}">
    <img src="./next-query-icon.svg" class="x-next-query__icon"/>
    <span class="x-next-query__query">{{ suggestion.query }}</span>
  </template>
</NextQueries>

Adding a custom next query component

You can use your custom implementation of a next query component. To work correctly, it should use the emitNextQuerySelected function when the next query is selected. In the example below, instead of using the default button tag for a next query, an icon is added, and the text of the next query is wrapped in a span

<NextQueries>
  <template #suggestion="{suggestion}">
    <NextQuery :suggestion="suggestion" class="x-next-queries__suggestion">
      <template #default="{suggestion}">
        <img src="./next-query-icon.svg" class="x-next-query__icon"/>
        <span class="x-next-query__query">{{ suggestion.query }}</span>
      </template>
    </NextQuery>
    <button>Custom Behaviour</button>
  </template>
</NextQueries>
